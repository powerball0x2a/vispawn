# 贡献指南

感谢你对 Vispawn 感兴趣！我们欢迎各种形式的贡献，包括代码、文档、问题反馈和模板贡献。

## 如何贡献

### 报告问题

如果你发现 bug 或有新功能建议：

1. 搜索 [Issues](https://github.com/powerball0x2a/vispawn/issues) 确保没有重复
2. 创建新 Issue，清晰描述问题或建议
3. 附上复现步骤和环境信息

### 提交代码

1. Fork 项目
2. 创建特性分支：`git checkout -b feature/amazing-feature`
3. 进行修改并提交：`git commit -m 'Add amazing feature'`
4. 推送分支：`git push origin feature/amazing-feature`
5. 创建 Pull Request

### 贡献模板

这是 Vispawn 社区生态的重要组成部分。通过贡献模板，你可以帮助其他用户更方便地创建可视化内容。

## 开发环境设置

```bash
# 克隆开发
git clone https://github.com/powerball0x2a/vispawn.git
cd vispawn

# 创建开发分支
git checkout -b dev

# 安装开发依赖
pip install -e .

python -m pytest
```

## 模板贡献指南

我们欢迎社区成员贡献可视化模板！通过贡献模板，你可以帮助其他用户更高效地创建原理可视化内容。

### 贡献流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  选择主题    │ →  │  编写模板   │ →  │  提交 PR    │ →  │  审核发布   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
     ↓                  ↓                  ↓                  ↓
  确定要可视化    按照规范编写    提交到 GitHub   社区审核通过后
  的原理内容      模板配置        仓库          合并到主分支
```

### 贡献前准备

1. **了解项目**：阅读 [快速开始](../get-started.md) 和 [API 参考](../api-reference.md)
2. **熟悉模板规范**：查看 [模板创建指南](templates/how-to-create.md)
3. **检查现有模板**：避免重复创建相似的模板
4. **测试环境**：确保本地开发环境可以正常运行

### 模板结构说明

每个模板是一个包含以下字段的字典：

```python
{
    "type": "geometric",        # 类型: geometric | mechanical | physical
    "description": "描述",       # 简短描述（20-50字）
    "concept": "可视化思路",      # 核心可视化概念（一句话说明）
    "elements": ["元素1"],       # 3D 元素列表
    "steps": ["步骤1"],          # 演示步骤（3-6个步骤）
    "audience": "middle",       # 目标受众: middle | high
    "complexity": "simple",     # 复杂度: simple | medium | complex
    "colors": {                 # 配色方案
        "primary": "#4a90e2",   # 主色 - 主要元素
        "secondary": "#e94560", # 辅色 - 次要元素
        "accent": "#ffd700",    # 强调色 - 重点标注
        "bg": "#f5f5f5"         # 背景色
    },
    "tags": ["标签"]            # 用于搜索（3-5个标签）
}
```

### 字段填写规范

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| type | 是 | 模板类型 | `geometric`, `mechanical`, `physical` |
| description | 是 | 原理简短描述 | "直角三角形两直角边的平方和等于斜边的平方" |
| concept | 是 | 可视化思路 | "在直角三角形三边上构建正方形，用方块填充展示面积相等" |
| elements | 是 | 3D元素列表 | `["直角三角形", "正方形", "面积方块"]` |
| steps | 是 | 演示步骤 | `["展示直角三角形", "构建正方形", "填充方块", "验证面积相等"]` |
| audience | 是 | 目标受众 | `middle`（初中）, `high`（高中/大学） |
| complexity | 是 | 复杂度 | `simple`（简单）, `medium`（中等）, `complex`（复杂） |
| colors | 是 | 配色方案 | 见下文 |
| tags | 是 | 搜索标签 | `["几何", "数学", "面积", "初中"]` |

### 颜色选择建议

选择对比明显、视觉舒适的颜色组合：

- **主色 (primary)**：主要展示元素，建议使用品牌色或主题色
  - 推荐：蓝色 `#4a90e2`、绿色 `#1abc9c`、紫色 `#9b59b6`

- **辅色 (secondary)**：次要元素，与主色形成对比
  - 推荐：红色 `#e94560`、橙色 `#e67e22`、青色 `#00d9ff`

- **强调色 (accent)**：重点标注、动画高亮
  - 推荐：黄色 `#ffd700`、亮青色 `#00ffff`

- **背景色 (bg)**：根据内容选择
  - 浅色背景：`#f5f5f5`、`#ffffff`
  - 深色背景：`#1a1a2e`、`#2c3e50`

### 标签选择规范

每个模板需要 3-5 个标签，便于搜索和分类：

- **学科标签**：数学、物理、化学、几何、力学
- **学段标签**：初中、高中、大学
- **内容标签**：面积、体积、传动、能量
- **难度标签**：（通过 complexity 字段体现）

### 提交 PR

1. 在 `vispawn/templates.py` 的 `TEMPLATES` 字典中添加新模板
2. 更新 `CATEGORIES` 分类索引（如适用）
3. 确保代码风格一致
4. 提交时添加清晰的提交信息：`feat(template): 添加XXX原理模板`
5. 在 PR 描述中说明：
   - 模板名称和类型
   - 简要说明可视化思路
   - 测试结果（如有）

### 审核标准

#### 会被接受的模板

- 原理正确、描述清晰
- 可视化思路合理、有教学价值
- 字段填写完整、规范
- 配色方案视觉舒适
- 标签准确、便于搜索

#### 常见拒绝原因

- 原理描述不正确或存在科学性错误
- 可视化思路不合理，难以理解
- 字段缺失或填写不规范
- 与现有模板高度重复
- 包含不当内容

### 质量要求

1. **科学准确性**：原理描述和可视化思路必须正确
2. **教学价值**：能够有效帮助学习者理解原理
3. **视觉质量**：配色协调、动画流畅
4. **用户体验**：步骤清晰、易于理解
5. **代码规范**：遵循项目代码规范

## 模板分类建议

我们欢迎各类原理的模板贡献，以下是推荐的分类：

### 几何类 (geometric)

适合展示形状、面积、体积等几何概念。

示例原理：
- 勾股定理
- 圆的面积
- 相似三角形
- 平行四边形性质

### 机械类 (mechanical)

适合展示机械原理、运动过程。

示例原理：
- 四冲程发动机
- 齿轮传动
- 活塞泵
- 杠杆原理

### 物理类 (physical)

适合展示物理定律、现象。

示例原理：
- 牛顿第二定律
- 电磁感应
- 光的折射
- 简谐运动

## 常见问题

### Q: 模板不会被 AI 覆盖吗？

预设模板优先于 AI 生成，确保核心原理的效果。当用户请求已存在模板的原理时，系统会优先使用预设模板。

### Q: 可以从用户生成的内容创建模板吗？

可以！用户生成的高质量内容经过审核后可以添加到模板库。

### Q: 模板支持自定义代码吗？

目前不支持，模板是纯配置。如需自定义代码，可以 fork 项目修改。

### Q: 如何知道模板是否被接受？

我们会在 PR 评论中反馈审核结果。如果需要修改，会提供具体建议。
